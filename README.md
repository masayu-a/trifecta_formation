# 三連複フォーメーション復元ツール

三連複の買い目一覧から、それらを生成できる三連複フォーメーションを逆算する Ruby プログラムです。

たとえば、次のような買い目一覧があるとします。

```text
3-10-12
3-10-18
3-12-16
3-12-18
...
```

このプログラムは、これらの買い目を完全に再現できるフォーメーション、たとえば

```text
12,18 - 10,12,16 - 3,8,10,12,13
```

のような形を探索します。

## 目的

三連複フォーメーションでは、

```text
1列目 - 2列目 - 3列目
```

の各列から1頭ずつ選び、重複しない3頭の組み合わせを買い目として生成します。

しかし、実際に購入後に残るのは個別の買い目一覧だけで、元のフォーメーションが分からなくなることがあります。

このツールは、買い目一覧から元のフォーメーション候補を復元するためのものです。

## 必要環境

Ruby 3.x 以降を想定しています。

標準ライブラリの `set` のみを使っているため、追加 gem は不要です。

## 使い方

Ruby ファイルを実行します。

```bash
ruby trifecta_formation.rb
```

プログラム内の `BET_TEXT` に買い目を記述します。

```ruby
BET_TEXT = <<~TEXT
3-10-12
3-10-18
3-12-16
3-12-18
TEXT
```

各行は次の形式で書きます。

```text
馬番-馬番-馬番
```

例：

```text
3-10-12
```

馬番の順序は問いません。

以下はすべて同じ買い目として扱われます。

```text
3-10-12
12-3-10
10-12-3
```

## 出力例

```text
元の買い目数: 19点

[1] 3,8,10,12,13 - 10,12,16 - 12,18
    マーク数: 10
    生成点数: 19点
```

列順を見やすく並べ替えると、次のように読めます。

```text
12,18 - 10,12,16 - 3,8,10,12,13
```

これは、

```text
1列目: 12,18
2列目: 10,12,16
3列目: 3,8,10,12,13
```

という三連複フォーメーションを表します。

## アルゴリズム

このプログラムは素朴な全探索でフォーメーションを探します。

手順は次の通りです。

1. 買い目に登場する馬番をすべて集める
2. その馬番集合から、空でない部分集合をすべて作る
3. それぞれの部分集合を、1列目・2列目・3列目の候補として試す
4. 各フォーメーションから生成される三連複の買い目集合を作る
5. 元の買い目集合と完全一致するものだけを解として採用する
6. 合計マーク数が少ない順に並べて表示する

たとえば、登場する馬番が

```text
3, 8, 10, 12, 13, 16, 18
```

であれば、これらの部分集合を列候補として使います。

```text
[3]
[8]
[3,8]
[10,12,18]
...
```

そのうえで、

```text
列1 - 列2 - 列3
```

のすべての組み合わせを試します。

## 三連複としての処理

三連複では、馬番の順序は関係ありません。

そのため、プログラム内部ではすべての買い目をソートして扱います。

```ruby
[12, 3, 10]
```

は、

```ruby
[3, 10, 12]
```

として保存されます。

また、同じ馬を複数回選ぶ組み合わせは無効です。

```ruby
[12, 12, 18]
```

のような組み合わせは除外されます。

## 解が複数出る理由

三連複フォーメーションには、同じ買い目集合を生成する別表現が複数あります。

たとえば、列の順番を入れ替えても、三連複では生成される買い目は同じです。

```text
A - B - C
C - A - B
```

また、列に余分な馬番を入れても、結果として新しい買い目が増えない場合があります。

そのため、このプログラムでは複数の解が出ることがあります。

出力では、次の基準で見やすい候補を上位に表示します。

1. 合計マーク数が少ない
2. 最大列サイズが小さい
3. 列サイズの並びが小さい
4. 馬番順

## 注意点

登場する馬番が増えると、探索量は急激に増えます。

馬番数を `n` とすると、空でない部分集合は

```text
2^n - 1
```

個あります。

3列の組み合わせをすべて試すため、単純にはかなり大きな探索になります。

少数の馬番からなる買い目の復元には十分ですが、多数の馬番を含む大規模な買い目では時間がかかる可能性があります。
