# trifecta_formation

`trifecta_formation` は、Trifecta（3連単）の買い目一覧から、それらを生成できるフォーメーションを逆算する Ruby プログラムです。

## Trifecta / 3連単とは

Trifecta（3連単）は、1着・2着・3着を着順通りに当てる馬券です。

たとえば、

```text
3-10-12
```

は、

```text
1着: 3
2着: 10
3着: 12
```

を意味します。

そのため、次の2つは別の買い目です。

```text
3-10-12
12-10-3
```

## 使い方

```bash
ruby trifecta_formation.rb
```

`trifecta_formation.rb` 内の `BET_TEXT` に買い目を記述します。

```ruby
BET_TEXT = <<~TEXT
3-10-12
3-10-18
8-10-12
TEXT
```

各行は次の形式です。

```text
1着-2着-3着
```

## 出力例

```text
元の買い目数: 8点

[1] 1着: 3,8 / 2着: 10,12 / 3着: 12,16,18
    マーク数: 7
    生成点数: 8点
```

## アルゴリズム

このプログラムは全探索でフォーメーションを探します。

1. 買い目に登場する馬番をすべて集める
2. その馬番集合から、空でない部分集合をすべて作る
3. それぞれを1着列・2着列・3着列の候補として試す
4. 各フォーメーションから生成される Trifecta（3連単）の買い目集合を作る
5. 元の買い目集合と完全一致するものだけを解として採用する
6. 合計マーク数が少ない順に並べて表示する

## Trio / 三連複版との違い

Trio（三連複）では、馬番の順序は関係ありません。

一方、Trifecta（3連単）では、着順が重要です。

```text
3-10-12
12-10-3
10-3-12
```

これらはすべて別の買い目です。

また、3連単では次の列順そのものに意味があります。

```text
1着候補 - 2着候補 - 3着候補
```

そのため、このプログラムでは列順を省略せずに探索します。

## 注意点

登場する馬番が増えると、探索量は急激に増えます。

馬番数を `n` とすると、空でない部分集合は次の個数になります。

```text
2^n - 1
```

3列についてすべて試すため、単純には次の通り数を調べます。

```text
(2^n - 1)^3
```

多数の馬番を含む大規模な買い目では時間がかかる可能性があります。
